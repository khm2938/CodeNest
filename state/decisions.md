# 결정 사항

왜 이렇게 했는지를 짧게 남기는 운영용 결정 기록이다.

## 2026-04-16 프롬프트와 상태 파일 분리

- 결정: 프롬프트와 변하는 프로젝트 상태는 분리한다.
- 이유: 반복 지침은 재사용하고, 현재 상태는 계속 갱신해야 컨텍스트 효율이 좋아진다.

## 2026-04-16 개발용 컨테이너는 하나로 시작

- 결정: 개발용 도커 컨테이너는 하나로 시작한다.
- 이유: 초반에는 복잡도를 낮추고, 나중에 필요할 때 서비스 단위로 확장하는 편이 더 안전하다.

## 2026-04-16 `PageRepository` 유지

- 결정: UI는 저장 방식과 직접 결합하지 않고 `PageRepository`를 통해 페이지를 읽고 쓴다.
- 이유: `localStorage`, API, DB 저장소를 갈아끼워도 화면 로직을 크게 바꾸지 않기 위해서다.

## 2026-04-16 SQLite + Prisma 선택

- 결정: 서버 저장소는 우선 SQLite + Prisma로 구현한다.
- 이유: 로컬 개발이 가볍고, 서버 API는 유지한 채 저장 계층만 교체하기 쉽다.

## 2026-04-16 한국어 UI 우선

- 결정: 사용자에게 보이는 문구와 작업 문서는 한국어를 기본으로 유지한다.
- 이유: 제품 방향과 작업 효율을 맞추고, 반복 작업에서 언어 혼선을 줄이기 위해서다.
