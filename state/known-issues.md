# 알려진 이슈

현재 확인된 버그와 깨진 흐름을 정리하는 파일이다.

## 열려 있는 이슈

### 2026-04-16 페이지 삭제 버튼 클릭 안정성

- 증상: 자동화 브라우저 QA에서 `페이지 삭제` 버튼이 인접 레이아웃 요소와 겹치면서 일반 클릭이 막히는 경우가 있다.
- 확인 방식: Playwright 클릭 시 `section-card__body`가 포인터 이벤트를 가로채는 상황을 재현했다.
- 영향: 삭제 기능 자체뿐 아니라 버튼 접근성과 레이아웃 안정성 점검이 필요하다.
- 다음 작업: 삭제 버튼이 다른 섹션에 가려지지 않도록 레이아웃과 클릭 경로를 확인한다.

## 해결한 이슈

### 2026-04-16 새 페이지 생성 직후 편집 대상 꼬임

- 증상: `새 페이지`를 누른 직후 바로 제목이나 본문을 수정하면, 방금 만든 페이지가 아니라 이전에 선택돼 있던 페이지에 첫 수정이 저장될 수 있었다.
- 원인: 생성 API가 끝나기 전까지 기존 페이지 편집기가 계속 활성 상태라서, 새 페이지가 준비되기 전 입력이 이전 페이지로 들어갈 수 있었다.
- 해결: 새 페이지 생성 중에는 편집기와 페이지 전환을 잠그고, 생성 완료 후에만 새 페이지를 편집할 수 있게 흐름을 바꿨다.
