# 블록 단위 편집 설계 문서

## 목표

CodeNest의 페이지 본문을 단일 textarea에서 블록 기반 편집기로 전환한다.
첫 버전은 개발자 특화 워크플로에 맞춰 `텍스트`, `제목`, `체크박스`, `코드블록` 4개 타입만 지원한다.

## 범위

### 포함

- 페이지 본문을 `blocks` 배열로 저장
- 블록 추가, 수정, 삭제
- 체크박스 토글
- 코드블록 언어 표시
- 실시간 미리보기
- 기존 페이지 목록, 페이지 제목 편집과 연결

### 제외

- 블록 드래그 앤 드롭 재정렬
- 중첩 블록
- 블록 링크
- 마크다운 자동 변환
- 외부 저장소 연동

## 데이터 모델

### PageItem

- `id: string`
- `title: string`
- `blocks: Block[]`
- `updatedAt: string`

### Block

- `id: string`
- `type: 'text' | 'heading' | 'checklist' | 'code'`
- `text: string`
- `checked?: boolean`
- `language?: string`

## 편집 규칙

- 페이지는 제목과 블록 목록으로 구성된다.
- 블록은 위에서 아래로 순서대로 렌더링된다.
- 각 블록은 독립적으로 수정할 수 있다.
- 체크박스 블록은 `checked` 값을 가진다.
- 코드블록은 `language` 값을 가진다.
- 블록 삭제는 즉시 반영한다.
- 블록 추가는 기본적으로 `text` 블록으로 시작한다.

## 화면 구조

### 왼쪽

- 페이지 목록
- 새 페이지 버튼

### 가운데

- 선택된 페이지의 블록 편집기
- 제목 입력
- 블록별 편집 UI

### 오른쪽

- 실시간 미리보기
- 현재 선택된 블록 타입 안내
- 블록 사용 도움말

## 구현 순서

1. `PageItem` 구조를 `blocks` 기반으로 변경
2. 블록 타입과 초기 데이터 추가
3. 기존 textarea 편집기를 블록 편집기 UI로 교체
4. 블록 추가/수정/삭제/토글 연결
5. 미리보기 렌더링 추가
6. 테스트 추가 및 기존 페이지 흐름 회귀 확인

## 에러 처리

- 선택된 페이지가 없으면 빈 상태를 보여준다.
- 블록이 비어 있어도 화면이 깨지지 않도록 기본 문구를 둔다.
- 코드블록 언어가 비어 있어도 `plain text`로 안전하게 표시한다.

## 테스트 범위

- 블록 타입별 데이터 생성이 올바른지 확인
- 새 페이지 생성 시 기본 블록이 들어가는지 확인
- 블록 내용 수정이 상태에 반영되는지 확인
- 체크박스 토글이 정상 동작하는지 확인
- 코드블록 언어 표시가 누락 없이 렌더되는지 확인

## 성공 기준

- 페이지 한 장 안에서 블록을 추가하고 수정할 수 있다.
- 제목, 체크박스, 코드블록이 모두 렌더된다.
- 새로 고침 전까지 현재 상태가 유지되는 기존 구조와 충돌하지 않는다.
- 이후 저장소 연동이나 블록 재정렬을 붙일 수 있는 구조로 남는다.
